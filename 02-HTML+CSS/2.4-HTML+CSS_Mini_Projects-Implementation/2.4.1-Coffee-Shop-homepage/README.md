# Coffee Shop Homepage - Implementation Challenge

## Project Overview

This is a **production-ready implementation challenge** where you'll build a complete coffee shop homepage demonstrating modern web development best practices, semantic HTML structure, responsive design, and perfect accessibility compliance.

## Challenge Requirements

### What You'll Build
- Professional coffee shop homepage with hero section
- Responsive navigation with mobile hamburger menu
- Featured products showcase
- Location and hours information
- Contact section with social media links

### Technical Standards
- **Pure HTML/CSS/JS**: No frameworks allowed
- **Mobile-First**: Responsive design starting from 320px
- **Accessibility**: WCAG 2.1 AA compliance required
- **Performance**: Page load under 3 seconds
- **Cross-Browser**: Chrome, Firefox, Safari, Edge support

### Key Learning Outcomes
- Semantic HTML5 structure implementation
- Mobile-first responsive design
- Accessibility best practices
- Performance optimization techniques
- Modern CSS layout methods

## Getting Started

### Prerequisites
- Basic HTML, CSS, and JavaScript knowledge
- Text editor or IDE
- Modern web browser
- Basic understanding of responsive design

### Setup Instructions
1. Create project directory: `coffee-shop-homepage/`
2. Set up file structure as specified in Goal.md
3. Start with mobile-first HTML structure
4. Implement CSS using progressive enhancement
5. Add JavaScript for interactive features

### Development Workflow
1. **HTML First**: Create semantic structure
2. **Mobile CSS**: Style for mobile devices first
3. **Desktop Enhancement**: Add larger screen styles
4. **JavaScript**: Add interactive functionality
5. **Testing**: Validate accessibility and performance
6. **Optimization**: Optimize images and code

## Testing Guidelines

### Accessibility Testing
- Use WAVE browser extension
- Test with screen reader (NVDA recommended)
- Verify keyboard navigation
- Check color contrast ratios
- Validate HTML markup

### Responsive Testing
- Test on mobile devices (320px+)
- Verify tablet layout (768px+)
- Check desktop experience (1024px+)
- Test orientation changes
- Validate touch interactions

### Performance Testing
- Use Lighthouse performance audit
- Optimize image sizes and formats
- Minimize CSS and JavaScript
- Test loading speed on slow connections
- Verify Core Web Vitals
