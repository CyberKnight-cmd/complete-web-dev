* Paragraphs (`p`)

## 151. Page Title Using `<h1>`

* `<h1>` = **most important heading**
* Typically used for:

  * Page title
  * Main topic

Example:

```html
<h1>Eels</h1>
```

üëâ One main topic per page is best practice.

---

## 152. Subheadings Using `<h2>`

* `<h2>` represents:

  * Secondary importance
  * Sections under the main topic

Example:

```html
<h2>About Eels</h2>
```

Hierarchy now:

* `h1` ‚Üí page title
* `h2` ‚Üí section title

---

## 153. Pro Tip: Close Tags Immediately

When writing longer code:

* Create opening **and closing tags first**
* Then fill in content

Example:

```html
<p></p>
```

Why:

* Prevents forgetting closing tags
* Reduces nesting errors

---

## 154. Adding Paragraph Content

Paragraphs provide:

* Explanations
* Descriptions
* Supporting text

Example:

```html
<p>Eels are fascinating aquatic creatures.</p>
<p>They have long, snake-like bodies.</p>
```

Each paragraph:

* Appears on its own line
* Is a block-level element

---

## 155. Using `<h3>` for Subsections

* `<h3>` = subsection under an `<h2>`

Example:

```html

```

Hierarchy now:

* `h1` ‚Üí Eels
* `h2` ‚Üí About Eels
* `h3` ‚Üí Eel Facts

## 156. Creating Lists in HTML

HTML supports **two types of lists**:

1. **Unordered list** (`ul`) ‚Üí bullets
2. **Ordered list** (`ol`) ‚Üí numbers

This lesson uses an unordered list.

---

## 157. Unordered Lists (`<ul>`)

* Creates a **bulleted list**
* Contains **list items**

Example structure:

```html
<ul>
  <li>Fact one</li>
  <li>Fact two</li>
</ul>
```

---

## 158. List Items (`<li>`)

* Each `<li>` = one bullet
* Must be placed **inside a list**

Example:

```html
<li>Eels can live in freshwater and saltwater.</li>
```

üëâ Browser automatically adds bullets.

## 160. Using Empty Elements: Horizontal Rule `<hr />`

* `<hr />` = horizontal separator line
* Used to:

  * Divide sections
  * Visually separate content

Example:

```html
<hr />
```

Key points:

* Empty element
* No closing tag
* Does not wrap content

---

## 161. Why `<hr />` Is Useful

* Breaks up long pages
* Improves readability
* Signals section changes

---

## 162. Nesting Inline Elements for Emphasis

You can combine inline elements.

Goal:

* Make **elver** bold **and** italic

Example:

```html
<strong><em>elver</em></strong>
```

---

## 163. Nesting Rule Applied (FILO Reminder)

Order:

1. Open `<strong>`
2. Open `<em>`
3. Close `</em>`
4. Close `</strong>`

üëâ **First In, Last Out**

## 165. Creating Links with Anchor Tags `<a>`

Links are created using:

* **Anchor element** `<a>`

Unlike earlier tags, anchor tags:

* **Require an attribute**

---

## 166. What Are Attributes?

* Attributes add **extra information** to elements
* Placed **inside opening tag**
* Format:

```
attribute="value"
```

---

## 167. The `href` Attribute (Most Important One)

* `href` = destination URL
* Required for anchor tags

Example:

```html
<a href="https://oceanconservancy.org">Ocean Conservancy</a>
```

---

## 168. Anatomy of an Anchor Tag

```html
<a href="URL">Link text</a>
```

* Attribute ‚Üí inside opening tag
* Value ‚Üí inside quotes
* Text between tags ‚Üí clickable

---

## 169. Common Anchor Tag Mistake (Exam Favorite)

‚ùå Forgetting to close the `<a>` tag:

```html
<a href="https://example.com">
Thanks for learning about eels
```

Result:

* Everything after becomes a link

---

## 170. Fixing the Anchor Tag Error

Correct:

```html
<a href="https://example.com">Ocean Conservancy</a>
<p>Thanks for learning about eels.</p>
```

Always:

* Close anchor tags
* Watch behavior carefully

---

## 171. What Happens When You Click a Link

* Browser navigates to the URL
* Current tab is replaced (by default)

This confirms:

* Anchor tag is working correctly
