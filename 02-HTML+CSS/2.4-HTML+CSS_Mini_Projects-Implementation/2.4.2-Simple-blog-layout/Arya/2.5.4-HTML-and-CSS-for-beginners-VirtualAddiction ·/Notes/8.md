## 190. Heading Hierarchy Best Practices

### Rule 1: One `<h1>` per page (or section)

* Represents:

  * Page title
  * Main topic

---

## 191. Rule 2: Maintain Logical Order

‚ùå Wrong:

```html

<h2>Sub-subheading</h2>
```

‚úÖ Correct:

```html

```

Hierarchy must flow **top ‚Üí down**.

---

## 192. Rule 3: Limit Heading Depth

* HTML provides **6 levels** (`h1‚Äìh6`)
* Best practice:

  * Use **no more than 3** per page when possible

Why?

* Deep hierarchies:

  * Hard to navigate
  * Confusing to users
  * Difficult for screen readers

---

## 193. Large Documents Strategy

If content requires:

* Many heading levels
* Deep nesting

Best solution:

* Split content across **multiple pages**
