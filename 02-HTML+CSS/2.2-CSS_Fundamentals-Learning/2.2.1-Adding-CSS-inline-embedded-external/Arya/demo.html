CSS Integration Methods Demonstration

This page demonstrates all three methods of adding CSS to HTML documents.

1. Inline CSS Example

Inline Styled Content

This content is styled using inline CSS with the style attribute. Notice how styles are applied directly to each element.

<div style="background-color: #ffebee; padding: 20px;"> <h3 style="color: #d32f2f;">Inline Styled Content</h3> <p style="color: #666;">This content uses inline CSS.</p> </div>

2. Embedded CSS Example

Embedded Styled Content

This content is styled using embedded CSS defined in the <style> tag within the document head.

<head> <style> .embedded-example { background-color: #e3f2fd; padding: 20px; border-left: 4px solid #2196f3; } </style> </head> <body> <div class="embedded-example">Content</div> </body>

3. External CSS Example

External Styled Content

This content is styled using external CSS loaded from a separate .css file.

<head> <link rel="stylesheet" href="external-styles.css"> </head> <body> <div class="external-example">Content</div> </body>

Method Comparison

Inline CSS

Pros:

  • Highest specificity
  • Quick testing
  • No HTTP requests

Cons:

  • Not reusable
  • Hard to maintain
  • Mixes content/presentation

Embedded CSS

Pros:

  • Page-specific styles
  • No HTTP requests
  • Can use selectors

Cons:

  • Not reusable
  • Increases HTML size
  • No caching

External CSS

Pros:

  • Reusable
  • Cacheable
  • Separates concerns
  • Easy maintenance

Cons:

  • Additional HTTP request
  • Render-blocking

Specificity Demonstration

This element has styles from all three methods applied:

  • Inline: Red border (highest specificity - wins)
  • Embedded: Blue background
  • External: Padding and margin

Best Practice Example

Recommended Approach

This section uses external CSS for all styling, demonstrating the recommended approach for production websites.

Why External CSS is Best:

  • Maintainable and scalable
  • Reusable across pages
  • Cacheable for performance
  • Separates content from presentation
