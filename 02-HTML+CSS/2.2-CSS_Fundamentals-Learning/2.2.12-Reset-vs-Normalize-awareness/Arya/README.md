# Reset vs Normalize Demo

## Setup Instructions
1. Open `demo.html` in any modern web browser
2. Compare the three sections showing different approaches
3. Notice styling differences between approaches

## Key Features Demonstrated
- **No Reset**: Browser default styling
- **CSS Reset**: All default styling removed
- **Normalize**: Useful defaults preserved, inconsistencies fixed

## Comparison Points
- **Headings**: Size and margin differences
- **Paragraphs**: Margin and spacing variations
- **Lists**: Bullet points and indentation
- **Overall appearance**: Visual consistency

## Expected Results
- No reset shows browser-specific defaults
- Reset removes all styling (elements look plain)
- Normalize maintains readability while ensuring consistency

## Decision Criteria
- **Use Reset**: When starting from scratch with complete design control
- **Use Normalize**: When building on existing patterns with accessibility needs
- **Use Custom**: Tailored approach combining both philosophies
