External Links

Secure External Links

External links should always include proper security attributes:

  • MDN Documentation
  • WCAG Guidelines
  • W3C Validator

Note: These links include rel="noopener noreferrer" for security.

Insecure External Link (Don't Do This)

This is how NOT to create external links:

<a href="https://external-site.com" target="_blank">Vulnerable Link</a>

Problem: Missing rel="noopener" creates security vulnerability.

Link Attributes Reference

Attribute Value Purpose Example
rel noopener Security: Prevents opener access Secure link
rel noreferrer Privacy: Hides referrer info Private link
rel nofollow SEO: Don't follow for ranking Untrusted link
target _blank Opens in new tab/window New tab
download filename Forces download with name Download as file.pdf

Accessibility Best Practices

Descriptive Link Text

Good examples of accessible link text:

  • Read our complete privacy policy
  • Download the user manual (PDF, 2.3MB)
  • View product specifications and pricing

Poor Link Text (Avoid These)

  • Click here for more information
  • Read more
  • Link

Problem: These don't provide context when read by screen readers.

Enhanced Accessibility

Links with additional context for screen readers:

  • External Resource (opens in new tab)
  • Annual Report 2024 PDF document, 5.2 megabytes
