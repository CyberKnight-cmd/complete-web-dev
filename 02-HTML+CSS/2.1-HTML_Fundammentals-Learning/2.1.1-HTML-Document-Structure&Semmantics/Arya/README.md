# HTML Document Structure & Semantics Demo

## Setup Instructions

1. Open `demo.html` in any modern web browser
2. Use browser DevTools (F12) to inspect the DOM structure
3. Test with accessibility tools for validation

## Running the Demo

### Basic Viewing
- Double-click `demo.html` to open in your default browser
- Observe the visual structure and semantic layout

### Testing Document Outline
1. Open browser DevTools (F12)
2. Go to Elements/Inspector tab
3. Notice the hierarchical structure of semantic elements
4. Use accessibility tree view if available

### Accessibility Testing
1. Install browser extension like axe DevTools
2. Run accessibility audit
3. Test with screen reader (if available)
4. Validate heading structure

## Key Features Demonstrated

- **Proper DOCTYPE**: HTML5 document type declaration
- **Semantic Structure**: header, nav, main, aside, footer
- **Content Sectioning**: article and section elements
- **Heading Hierarchy**: Logical h1-h3 progression
- **Text Semantics**: em and strong elements
- **Accessibility**: Proper landmark roles and structure

## Validation Steps

1. **W3C Validator**: Copy HTML and paste into https://validator.w3.org/
2. **Outline Check**: Use HTML5 Outliner tool
3. **Screen Reader Test**: Navigate using heading shortcuts
4. **DevTools Audit**: Run Lighthouse accessibility audit

## Expected Results

- Valid HTML5 markup
- Clear document outline
- Accessible navigation structure
- Proper semantic meaning for all content
