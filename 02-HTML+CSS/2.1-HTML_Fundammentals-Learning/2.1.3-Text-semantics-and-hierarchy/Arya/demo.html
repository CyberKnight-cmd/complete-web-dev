Text Semantics & Hierarchy Demonstration

This page demonstrates proper HTML text semantics and heading hierarchy with practical examples.

Heading Hierarchy Examples

Correct Heading Structure

This demonstrates proper heading hierarchy without skipping levels:

Article: "Understanding Web Development"

Section: Introduction to HTML
Subsection: HTML Elements
Subsection: HTML Attributes
Section: CSS Fundamentals
Subsection: Selectors
Subsection: Properties

Incorrect Heading Structure (Don't Do This)

This shows what not to do - skipping heading levels:

<h1>Main Title</h1>
<h4>Section</h4> <!-- WRONG: Skipped h2 and h3 -->

Text Semantic Elements

Emphasis vs Strong Importance

I really love this chocolate cake! (stress emphasis)

I really love this chocolate cake! (different emphasis, different meaning)

Warning: This action cannot be undone. (strong importance)

The meeting is today at 3 PM. (urgent/important)

Code Examples

Inline Code

Use the console.log() function to output messages in JavaScript.

The HTML <div> element is a generic container.

Code Blocks

Here's a JavaScript function example:

function calculateArea(radius) {
    const pi = 3.14159;
    return pi * radius * radius;
}

// Usage example
const area = calculateArea(5);
console.log("Area:", area);

Keyboard Input

Press Ctrl + S to save your work.

Use F12 to open browser developer tools.

Sample Output

When you run the above function, you'll see:

Accessibility Benefits

Proper text semantics and hierarchy provide several accessibility benefits:

  • Screen Reader Navigation: Users can jump between headings using shortcuts
  • Document Structure: Assistive technologies understand content organization
  • Search Engine Optimization: Better content understanding and indexing
  • Keyboard Navigation: Logical tab order and focus management
